# Codex 执行任务书：政策法规日报

本文件是项目持续实施和验收的规范基线。详细原始任务书由用户于 2026-07-22 提供；以下要求均为强制要求。

## 目标与运行计划

开发并部署“政策法规日报”，监测全球重卡，重点关注纯电重卡、动力电池、充换电、排放、碳排放、智能驾驶及市场准入。每天北京时间 08:30 采集此前 24 小时可核验新增信息，使用 DeepSeek 分类、翻译和生成中文摘要，生成并发布日报；有有效内容时才允许通过 QQ SMTP 邮件发送。每周一 08:30 汇总此前 7 天，每月 1 日 08:30 汇总上个自然月，周报和月报只发布网站、永不发送邮件。

目标仓库为公开的 `qesbr/heavy-truck-policy-daily` 和私有的 `qesbr/heavy-truck-policy-daily-config`，网站地址为 `https://qesbr.github.io/heavy-truck-policy-daily/`。所有 GitHub Actions 时间使用 UTC，日报 `30 0 * * *`，周报 `30 0 * * 1`，月报 `30 0 1 * *`。手动触发支持报告类型、起止日期、是否发送日报邮件及强制重建。

## 监测范围与来源

重点地区：中国、欧盟、UNECE/WP.29、沙特阿拉伯、伊朗。一级分类只能是政策、法规、标准、市场、企业之一。固定标签为纯电重卡、动力电池、充电、换电、排放、碳排放、智能驾驶、市场准入；标签需要同义归并，每条最多展示三个且优先保留核心标签。

采用白名单优先加可靠来源发现。首批必须覆盖 UNECE/WP.29、欧委会与 EUR-Lex、工信部、交通运输部、生态环境部、国家发改委、国家标准委、全国标准信息公共服务平台、全国汽车标准化技术委员会、中国汽车工业协会、中国汽车工业信息网、ICCT、中国汽车报及卡车之家。微信公众号白名单按用户原始任务书配置；只能使用合法公开入口，独立适配层失败不得影响官方来源。首期不部署 WeRSS 常驻服务，不接入收费 API。

每个来源保存名称、网址、类型、首次发现、最近成功采集时间和采集状态。新增候选未通过可靠性及正文规则前不得自动进入正式报告。

## 收录、AI 与去重规则

只有正文可访问、实际发布时间可确认且处于采集窗口、主题相关、原文链接有效、来源真实且摘要受正文支撑的信息可以收录。只有标题或搜索片段、正文不可访问、无法确认日期、链接失效、弱相关、低可信或 AI 输出校验失败时必须跳过。禁止依据标题或搜索摘要推断全文。

DeepSeek 严格 JSON 输出并通过数据模型验证，完成相关性、中文翻译与标题、150—300 字中文摘要、单一主分类、地区、标签、0—100 重要性评分和同事件判断。摘要只包含核心内容、主要变化、适用对象和明确时间节点，不得提供行动或经营建议，不得增加原文结论。外文记录显示中英文标题；中文记录不重复原文标题。

至少实施 URL 规范化、标题与正文相似度、AI 同事件三层去重。同事件以法规原文、政府或标准组织等最权威来源为主卡片，其余来源折叠为相关报道并保留标题、来源和原文链接。

## 网站、数据、邮件与安全

网站包含首页、日报/周报/月报归档、单期报告和来源状态，支持报告切换、日期归档、全文搜索，以及分类、地区、信息类型、来源、日期和标签筛选。同事件支持多来源折叠；桌面和移动端自适应；具有加载、空数据和错误状态。所有路径兼容 GitHub Pages 项目子路径。

数据采用版本化 JSON，保留原始、处理后、日报、周报、月报、来源状态和搜索索引。信息内部字段至少包括稳定 ID、中英文标题、摘要、来源、发布时间、采集时间、地区、主分类、标签、重要性、重点标记、内容哈希、事件 ID 和相关来源，确保可追溯和重复运行幂等。

QQ SMTP 为 `smtp.qq.com:465`。敏感信息仅从 Secrets 或环境变量读取，包含 `DEEPSEEK_API_KEY`、`SMTP_USERNAME`、`SMTP_AUTH_CODE`、`PRIVATE_CONFIG_TOKEN`、`PRIVATE_CONFIG_REPO`。私有令牌只读目标配置仓库。邮件适配手机，发送失败不得阻断网站，日志不得泄漏邮箱或凭据。未经用户明确同意，不得发送真实测试邮件。

使用 Python 3.11、插件化采集器、合理超时/重试/限速和清晰 User-Agent；遵守公开访问规则，不绕过登录、验证码或访问控制。对外部内容清洗防止 XSS，日志脱敏，固定关键依赖，Actions 最小权限。

## 验收

必须完成单元与集成测试、配置校验、DeepSeek 模拟和错误测试、采集失败隔离、幂等、有/无内容日报、周月报、邮件预览与安全、静态构建、Pages 子路径、桌面和移动端、链接/搜索/筛选/归档检查。最终交付必须区分本地代码完成与 GitHub 实际部署，报告测试结果、网站和两个仓库地址、Secrets 清单、首次 Actions 运行结果及已知限制。
